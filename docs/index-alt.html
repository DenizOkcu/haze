shape

Everything is plain and inspectable.

Project instructions are files. Skills are Markdown. Sessions are resumable. Diffs are still your problem.

  • Does Runs in your terminal. Keeps file tools in the workspace. Runs bash and summarizes checks.
  • Context files Reads AGENTS.md and CLAUDE.md as project instructions.
  • Sessions Supports resumable sessions, compaction, and continuing work in the current repo.
  • Skills Skills are Markdown. Use them for procedures you do more than once.
  • Subagents Supports subagents for separate investigations when parallel work actually helps.
  • Does not Hide tool calls. Ship a dashboard. Add popup rituals. Pretend to be magic. Make you watch a product tour.

tools

Seven tools. Clear output.

Enough to let the model work. Small enough to see what it did.

  • Read Read files from the current workspace with line ranges when needed.
  • Search Search the repo with ripgrep-backed patterns instead of reading files one by one.
  • Edit Use targeted replacements or line-range edits for existing files.
  • Write Create files when the task needs new source, tests, docs, or skills.
  • Bash Run commands, tests, typechecks, builds, and summarize recognized validation output.
  • List Inspect the project tree without falling back to shell discovery for everything.
  • Replace lines Use line ranges when exact text edits are ambiguous.

does not

Haze does not hide tool calls, ship a dashboard, add popup rituals, pretend to be magic, or make you watch a product tour.

faq

Common questions.

  • What is Haze?

    Haze is a small agentic coding tool for the terminal. It reads, edits, searches, writes, and runs commands. Every call is visible.

  • How do I install and run Haze?

    Install with npm i -g @denizokcu/haze, then cd into a repo and run haze. Ask for one change. If the result is useful, keep going.

  • Do I need a paid AI subscription to use Haze?

    No. Haze is the tool. You still configure the model provider you want to use.

  • Can I use Haze with my existing AI provider?

    Yes. Haze uses OpenAI-compatible providers. Configure one in settings or environment variables, then switch in-session with /provider and /model.

  • Is Haze terminal-only?

    Yes, by design. Haze is a CLI tool. There is no desktop app or IDE extension. The terminal is the product.

  • How much does Haze cost?

    Haze is free and MIT-licensed. Model usage depends on the provider you configure.

  • What about my code and privacy?

    Haze uses local files and local session state. Prompt context goes to the model provider you configure.

  • Is Haze open source?

    Yes. Haze is an MIT-licensed TypeScript package published as @denizokcu/haze.

  • What are Markdown skills?

    Skills are Markdown. Use them for instructions you want to reuse instead of retyping.
