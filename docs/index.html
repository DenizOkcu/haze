// getting started

Install. Add a provider.
Pick a model.

Haze starts intentionally small. First give it an OpenAI-compatible provider, then select the model it should use.

  1. 01$ hazeOpen Haze from a project directory after installing.
  2. 02/providerChoose OpenRouter or add any OpenAI-compatible endpoint with URL, optional key, and models.
  3. 03/modelPick the first model, or set one directly like /model local:llama3.1.
  4. 04Start chattingAsk Haze to inspect, edit, run commands, or explain the project.
  5. 05/create-skillCreate Markdown skills in ~/.haze/skills for reviews, release prep, deploy checks, or your own workflows.

That's the core on purpose. Anything more specific is meant to be created with the LLM through skills.

// what it does

Six tools. One agent.
Infinite stubbornness.

Haze doesn't try to be everything. It gives the model read, write, edit, list, and bash — then watches it figure things out.

Born in the terminal

No web UI. No Electron. No Docker container that spawns other Docker containers. Haze lives in your terminal where code happens. Interactive TUI with streaming, syntax highlighting, and tool-call transparency. Watch it work. Keep your hands near the wheel.

Deliberately small toolset

Read, edit, write, list, bash. That's it. No 47-tool sprawl. No tool that scaffolds a tool that generates a plan that plans to generate a plan. The model has six tricks. It's surprisingly effective.

Markdown skills

Notice a pattern? Run /create-skill and teach Haze a new trick. Skills are just Markdown files. No YAML schemas, no plugin APIs, no registry. Write a prompt, save it, use it. Tiny spell, useful goblin.

Context-aware

Reads AGENTS.md and CLAUDE.md from your project and home directory. Durable sessions can be resumed with haze --continue, and /compact keeps long chats manageable. Rude, but in a good way.

Light guardrails

File tools scoped to your workspace. Gitignore-aware by default. Bash mutations discouraged. Destructive actions require confirmation. Powerful enough to help, dumb enough to deserve supervision.

Bring your own model

Works with any OpenAI-compatible API via /provider, /model, or environment variables. Grok, GPT, Claude, DeepSeek, your weird local llama — if it speaks the protocol, Haze speaks to it. Default: x-ai/grok-build-0.1.

// the magic system

Skills grow while you work.

Hit friction? Create a skill. Keep going. The most adaptive workflow is the one you shape as you go.

/create-skill
Describe what you want. Haze writes the Markdown. You refine it. Done.
/deploy-check
Your custom skill, a slash command now. Lint, test, build, check. One invocation.
/review
Teach it your review standards. It remembers next time. Every time.
~/.haze/skills/
Plain Markdown files. Edit them. Delete them. Version them. They're yours.
